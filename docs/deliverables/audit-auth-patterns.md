## 3. Audit Questions (Q1-Q10)

### Q1. Login đang xảy ra ở app, gateway hay cả hai?

- WordPress: Cả hai. OpenIG runs OIDC at `/openid/app1`, then `CredentialInjector.groovy` posts credentials to `http://wordpress/wp-login.php` and caches upstream cookies (`stack-a/openig_home/config/routes/01-wordpress.json:33-47`, `stack-a/openig_home/scripts/groovy/CredentialInjector.groovy:46-97`).
- WhoAmI: Gateway only. OpenIG runs OIDC at `/openid/app2` and forwards identity in `X-Authenticated-User`; there is no app login flow in scope (`stack-a/openig_home/config/routes/02-app2.json:33-76`).
- Redmine: Cả hai. OpenIG runs OIDC, then `RedmineCredentialInjector.groovy` does `GET /login` plus `POST /login` to Redmine and caches `_redmine_session` (`stack-b/openig_home/config/routes/02-redmine.json:87-101`, `stack-b/openig_home/scripts/groovy/RedmineCredentialInjector.groovy:27-188`).
- Jellyfin: Cả hai. OpenIG runs OIDC, then `JellyfinTokenInjector.groovy` calls `POST /Users/AuthenticateByName` and injects the issued token (`stack-b/openig_home/config/routes/01-jellyfin.json:92-114`, `stack-b/openig_home/scripts/groovy/JellyfinTokenInjector.groovy:48-107`).
- Grafana: Gateway only. OpenIG runs OIDC and forwards `X-WEBAUTH-USER` to Grafana auth proxy (`stack-c/openig_home/config/routes/10-grafana.json:33-47`, `stack-c/openig_home/config/routes/10-grafana.json:76-88`, `stack-c/docker-compose.yml:57-61`).
- phpMyAdmin: Cả hai. OpenIG runs OIDC, fetches credentials from Vault, then injects HTTP Basic on requests to phpMyAdmin (`stack-c/openig_home/config/routes/11-phpmyadmin.json:33-47`, `stack-c/openig_home/config/routes/11-phpmyadmin.json:84-97`, `stack-c/openig_home/scripts/groovy/VaultCredentialFilter.groovy:128-157`).

- UNVERIFIABLE for all six apps from static code alone. The code proves that WordPress and Redmine expect new `Set-Cookie` values on successful upstream login, but it does not prove whether a pre-auth session ID is rotated versus newly created (`stack-a/openig_home/scripts/groovy/CredentialInjector.groovy:82-97`, `stack-b/openig_home/scripts/groovy/RedmineCredentialInjector.groovy:143-188`).
- For WhoAmI and Grafana, no app-local session cookie is shown in the inspected code; only OpenIG OIDC state is visible (`stack-a/openig_home/config/routes/02-app2.json:33-76`, `stack-c/openig_home/config/routes/10-grafana.json:33-88`).
- For Jellyfin and phpMyAdmin, the code shows token/basic-auth injection, not a before/after browser cookie trace (`stack-b/openig_home/scripts/groovy/JellyfinTokenInjector.groovy:57-107`, `stack-c/openig_home/config/routes/11-phpmyadmin.json:84-97`).
