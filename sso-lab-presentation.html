Bài toán thực tế

Hiện trạng

  • Mỗi ứng dụng legacy có cơ chế login riêng — Form, Basic, Header, Token, LDAP
  • Người dùng phải nhập mật khẩu nhiều lần khi truy cập các hệ thống khác nhau
  • Không có cơ chế đăng xuất tập trung — logout 1 app, các app khác vẫn giữ session
  • Credentials phân tán, khó quản lý, khó audit

Thách thức

  • Ứng dụng legacy không thể sửa code — vendor lock-in, out of support, thiếu source
  • Mỗi app dùng framework/protocol hoàn toàn khác nhau
  • Cần giải pháp không xâm lấn (non-intrusive) — không động vào application layer
  • Vẫn phải đảm bảo security standards — revocation, secret management, HA

06
SSO Flow

Luồng đăng nhập

1
User truy cập app → nginx forward tới OpenIG (ip_hash sticky)
2
OpenIG check JwtSession cookie → chưa có → redirect sang Keycloak login
3
User đăng nhập Keycloak → OIDC callback → OpenIG nhận id_token + access_token
4
OpenIG gọi Vault lấy credentials cho app đích (AppRole → per-user secret)
5
Adapter filter inject credentials vào app theo cơ chế phù hợp (Form/Basic/Header/Token)
6
App phản hồi → OpenIG tạo JwtSession cookie → trả browser. Lần sau: session tự động

User chỉ đăng nhập 1 lần — truy cập app khác tự động SSO qua Keycloak session

07
SLO Flow

Luồng đăng xuất

RP-Initiated

1
User click logout → OpenIG SloHandler intercept
2
SloHandler build end_session_endpoint URL + id_token_hint
3
Redirect browser → Keycloak xóa SSO session

Backchannel

4
Keycloak gửi logout_token (JWT RS256) tới mỗi OpenIG
5
BackchannelLogoutHandler validate JWT → extract sid
6
Ghi blacklist:sid vào Redis (TTL = session timeout)

Mỗi request tiếp theo → SessionBlacklistFilter check Redis → nếu blacklisted → kick session → yêu cầu đăng nhập lại

Security

7 Required Controls

1
Revocation Contract — Redis TTL ≥ session timeout, fail-closed 500
2
Secret Externalization — Vault runtime, không hardcode
3
Transport Security — HTTPS, Secure cookie, requireHttps
4
Session Storage — Không lưu secrets trong JwtSession
5
Pinned Origins — Static redirect URLs, không derive từ Host header
6
Bounded Dependencies — Socket timeout 200/500ms, error code chuẩn
7
Adapter Contract — Filter chain verified end-to-end, fail-closed

Derived từ code review + security review 3 stacks — tài liệu hóa trong Standard Gateway Pattern v1.1

09
Infrastructure

Vault + Redis

Vault — Secret Management

  • File storage — persist data, production-grade (không dev mode)
  • AppRole auth — role_id + secret_id, TTL 72h rotation
  • Per-user credentials — mỗi SSO user map tới app credentials riêng
  • Audit logging — mọi access đều được ghi log
  • KV max_versions=5 — secret versioning cho rollback

Redis — Revocation Store

  • Session blacklist — key: blacklist:sid, value: "revoked"
  • TTL = session timeout (1800s) — tự cleanup khi session hết hạn
  • appendonly yes — persist blacklist survive restart
  • Fail-closed — Redis down → trả 500, không proxy unauthenticated
  • Socket timeout — connect 200ms, read 500ms → bounded latency
